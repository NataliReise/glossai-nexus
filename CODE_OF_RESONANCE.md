# Code of Resonance

This document is an early charter for **glossai-nexus**.

It describes the spirit of the project rather than a complete set of rules.

## 1. A Nexus should invite response

A Nexus is not meant to overwhelm, manipulate, or replace human judgement.

It should open a space in which people can play, read, question, answer, and connect.

## 2. The code should speak

A Nexus is an open software artifact. Its code, structure, and rules should be readable and inspectable.

Speaking code has three layers:

- **Clear code:** the functional code should be understandable, maintainable, and well documented.
- **Playful code:** comments, names, traces, notes, configuration patterns, and Git history may carry optional game meaning.
- **Ethical code:** the architecture should reflect accessibility, non-elitism, openness, privacy of activations, and human responsibility.

A core principle:

> The code belongs in the light. The message may remain protected.

Another core principle:

> Speaking code should be clear before it becomes playful.

## 3. Structure and relationship are different

The shared module structure should be open.

Personal activations may remain private.

Names, personal messages, gifts, individual responses, and private contexts do not have to be published for a module to count as open.

## 4. Open does not mean elitist

Players should not be forced to read or understand code.

A Nexus should be playable at the surface and readable in the depth.

> You do not have to read the code. But the code must be readable.

## 5. Forks are resonance forms

Forks, variations, and alternative implementations should not be treated as failures by default.

They can become new resonance forms with visible ancestry.

## 6. Power should be inspectable

A Nexus should make its rules, assumptions, and technical behavior as transparent as possible.

Hidden power is not resonance.

## 7. AI assistance needs human responsibility

AI may support the project, but responsibility remains human.

A Nexus should not outsource judgement, authorship, care, or ethical decisions to a machine.

## 8. Democracy and human dignity matter

The project is interested in communication, cooperation, democratic imagination, and human dignity.

A Nexus should help create spaces where difference, conflict, and uncertainty can become readable and discussable rather than simply erased.

## 9. The network grows through carried traces

A Nexus should not let systems speak behind people's backs.

Its core play experience should work without an internet connection. If activations connect, they should do so through traces that people choose to carry: resonance keys, shared files, returned codes, pull requests, Hall of Resonance entries, or other voluntary acts of exchange.

Nothing should be tracked, uploaded, or transmitted silently.

The network grows when people choose to carry a trace.
