# What next after First Spark?

You have completed First Spark.

You do not have to do anything else.

First Spark can simply remain a finished gift: something you opened, read, and may return to later.

## What did I just play?

First Spark is a small local Nexus module.

It can be played as a neutral public demo or as a private gift when it is combined with a private activation package.

During the playthrough, you:

1. entered the First Spark chamber,
2. read two public traces,
3. linked the spark,
4. unlocked the activation message,
5. saw an after-play note with optional next steps.

The public repository contains only the clean public module, safe example data, and public-safe documentation.

Private activation files, private gift messages, private recipient data, and return artifacts do not belong in the public repository.

## What can I do now?

You may simply keep First Spark as a completed gift.

Nothing else is required.

If you want, you can let the spark travel further:

1. Share the clean public repository or a clean public copy of First Spark.
2. Add private activation data only through a private channel.
3. Never publish private activation files, private gift messages, or return artifacts.

Passing on a spark is an invitation, not an obligation.

## What is a resonance node?

A resonance node is an optional public-safe trace.

It can show that a First Spark was seen or completed without revealing private content.

In the terminal, after completing First Spark, type:

```text
resonance-node
```

First Spark will show a draft that you can copy manually.

Only edit the public alias and public note unless you are sure the other fields still remain public-safe.

A public note can be very small, for example:

```text
A First Spark was seen here.
```

or:

```text
The spark arrived, was opened, and may travel further.
```

Do not include the private activation message in a resonance node.

## How can I share a resonance node?

Sharing a resonance node is optional.

If you want to share one:

1. Type `resonance-node` in First Spark after completion.
2. Copy the draft from the terminal.
3. Edit only the public alias and public note.
4. Open the project discussions.
5. Post it in the `Resonance Nodes` category.

Project discussions:

```text
https://github.com/NataliReise/glossai-nexus/discussions
```

Before posting, check that the text contains no private activation message, no private file content, and no return artifact.

## Why does First Spark not save a result file?

First Spark does not automatically save result files, resonance nodes, return artifacts, or play logs.

This is intentional.

If you want to keep or share something, copy the visible terminal text manually.

This keeps the module simple and reduces accidental data creation.

## What must stay private?

Do not publish:

- `activation.local.json`
- private gift messages
- recipient-specific private data
- return artifacts
- private notes from the gift package

## What is safe to share publicly?

Usually safe:

- the public repository link
- a neutral public copy of First Spark
- a public-safe resonance node draft
- a short public note that does not reveal private gift content

When unsure, leave it out.
