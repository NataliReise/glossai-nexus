## Release blockers and minimum negative path

A confirmed finding is a release blocker when it materially:

- prevents the binding gift path;
- defeats or seriously misrepresents the gift objective;
- violates the canonical technical contract;
- violates a privacy, safety, or publication boundary;
- gives materially false or unusable release instructions; or
- leaves the handoff package different from the package actually accepted.

The minimum negative-path verification includes:

- COMPOSE cancellation;
- ANSWER cancellation;
- nonproductive BLOCKED behavior;
- missing Known Source;
- invalid Known Source;
- zero Return candidates;
- multiple Return candidates without automatic selection;
- repeated Opening without overwrite;
- second independent COMPOSE cycle;
- no discovery, automatic repair, regeneration, or transfer.
