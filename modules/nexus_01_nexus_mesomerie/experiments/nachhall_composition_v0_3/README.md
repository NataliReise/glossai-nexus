# Nachhall Composition V0.3 Microprototype

Status: active experiment

Production use: no

Current direction source: `../../CURRENT_DIRECTION.md`

## Purpose

This experiment tests whether the compact **Nachhall** form can become the complete poetic resonance result of Nexus 01.

The form is:

```text
2 / 4 / 6 / 4 / 1
```

The experiment deliberately removes the requirement to compose a long-form Resonance Artifact before producing the final poem.

## First test world

```text
image      open starry window
scent      summer rain
movement   returning tide
```

The first microprototype should remain narrow enough to inspect by hand while already testing the main design idea:

```text
small curated profile field
+ several micro-routes
+ curated phrase variants
+ two free words
= many compact valid Nachhalle
```

## Core principles

- The Nachhall is the complete resonance form.
- The poem may be enigmatic but should retain an internal movement.
- At least one concrete or sensory anchor should remain visible.
- One controlled metaphorical leap is desirable.
- Chamber selections shape permissions and motifs rather than forcing direct quotation.
- Wish and return words receive no semantic analysis.
- Phrase variants are curated as complete compatible units before any finer word-slot system is considered.
- Strong fixed lines may remain available as rare forms.
- Local randomness must be injectable or seedable for tests.
- A production result will later be created once and reopened unchanged.

## Structural contract under test

```text
line 1   2 words   image seed or atmospheric threshold
line 2   4 words   opening relation atmosphere or early wish placement
line 3   6 words   central movement relation or wish placement
line 4   4 words   answering trace remainder or late wish placement
line 5   1 word    return word
```

Micro-routes may place the wish word in line 2, 3, or 4.

The wish word must appear exactly once.

The return word must appear exactly once as line 5.

## Initial micro-route sketches

### Route A — moving wish

```text
image seed
opening or atmosphere
wish word in movement
answering trace
return word
```

### Route B — early wish

```text
image seed
wish word in opening
shared movement or relation
remaining trace
return word
```

### Route C — late wish

```text
image seed
atmosphere or threshold
relation or shared movement
wish word in remainder
return word
```

These are working sketches, not frozen route names or schemas.

## Variant strategy

Preferred first mechanism:

```text
curated complete phrase variants
```

Example principle:

```text
follows the distant light
crosses the rainlit threshold
enters the returning water
```

The first version should not independently choose arbitrary verbs, objects, and complements. Grammatical compatibility alone is not enough; each phrase must preserve a reviewed poetic operation.

## Productive mystery

The poem may leave meaning open, but it should not become arbitrary darkness.

A useful review rule is:

```text
one sensory anchor
+ one perceivable movement
+ one surprising metaphorical relation
+ no more than one or two strong semantic leaps
```

## First acceptance criteria

A first executable microprototype should demonstrate:

- exact word counts in all five lines;
- both free-word placement rules;
- no punctuation dependency;
- no semantic word interpretation;
- at least three structurally distinct micro-routes;
- multiple reviewed phrase variants per highly visible role;
- reproducible output under a supplied seed;
- more than one valid surface for the same selections and words;
- calm failure when no valid route is available;
- a small human-review corpus including ordinary, unusual, German, and identical words.

## Explicit exclusions

This first experiment does not need:

- a long-form Resonance Artifact;
- seven-role long-form assembly;
- a universal grammar engine;
- free independent verb/object recombination;
- all 125 Chamber combinations;
- production persistence;
- V0.1 or V0.2 archive movement;
- migration of existing saved results;
- a final universal metadata schema.

## Relationship to earlier systems

V0.1 provides the original Nachhall form, strict validation, and several successful complete examples.

V0.2 provides profile ideas, local weighting, anti-formula findings, Same-Word observations, and evidence about where long-form complexity becomes disproportionate.

This experiment keeps those lessons while changing the visible target:

```text
not long form then Echo
but one complete compact Nachhall
```

## Next work unit

Before implementation, define a very small first library for the test world:

- 3 to 5 variants for line 1;
- 4 to 6 variants for each line-2 role;
- 5 to 8 variants for each line-3 role;
- 4 to 6 variants for each line-4 role;
- three micro-routes;
- explicit compatibility and rarity notes only where needed.

The first goal is not maximum combinatorics.

The first goal is a small family of Nachhalle that feel related, varied, and worth rereading.
