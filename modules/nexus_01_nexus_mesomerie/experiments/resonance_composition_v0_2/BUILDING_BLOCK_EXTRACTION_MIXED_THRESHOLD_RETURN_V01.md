# Building-Block Extraction for the Mixed Threshold and Return World V0.1

This document explains the first extraction of a small poetic building-block library from `MANUAL_PAIRS_MIXED_THRESHOLD_RETURN_V01.md`.

The accompanying machine-readable experiment is:

```text
building_blocks.mixed_threshold_return.prototype.json
```

Neither file changes the V0.1 production path.

---

## 1. Purpose of this extraction

The manual pairs demonstrated that `converging-return` can support several distinct poetic forms:

- approach without fusion;
- contact before meeting;
- shared direction;
- a held-open interval;
- return without undoing departure.

The next question is whether those discoveries can be converted into a small transparent library without reducing them to five new fixed poems.

The extraction therefore separates:

```text
poetic function
from
one complete poetic work
```

A complete manual pair is never selected as one production object.

Instead, the prototype selects reviewed blocks for field, atmosphere, free-word entry, movement, response, relation, and remainder.

---

## 2. What was preserved from the manual experiments

The library preserves structural discoveries that appeared repeatedly or carried clear value:

```text
an open threshold can hold distance without sealing it
rain can soften or reveal a boundary
Hope may enter as a figure in motion
return may preserve earlier departure
a second current may answer without absorbing the first
two movements may share orientation without fusion
an interval may remain open as the relational result
Spirit may occupy the remaining passage rather than solve Hope
```

These are encoded through:

- role;
- profile requirements;
- operators;
- resonant gains;
- remaining states;
- Completion Mode compatibility;
- terminal permission;
- Echo eligibility.

The generator does not understand any of these ideas. It matches tags and follows reviewed transition rules.

---

## 3. What was deliberately not extracted

Several attractive manual lines remain one-off poetic discoveries for now.

They were not converted directly into blocks when they were:

- too complete to combine without dominating a new poem;
- strongly tied to one particular surrounding passage;
- more explanatory than imagistic;
- dependent on ambiguous pronouns;
- likely to produce a conspicuous repeated GlossAI mannerism.

Examples of structures not promoted as direct first-library blocks include:

```text
one carries the other only through shared direction

Spirit becomes the quiet direction they share

neither motion names the place of meeting
```

Their underlying ideas may remain present through tags such as `sharing-direction`, `continued-separateness`, or `held-open-relation`.

This follows the same principle already adopted for the older V0.1 texts:

```text
A formulation may disappear
Its poetic force may remain
```

---

## 4. Size of the first library

The first library is intentionally small.

It contains:

```text
3 structural routes
3 field blocks
3 atmosphere blocks
4 wish-entry blocks
3 movement blocks
3 response-movement blocks
4 relation blocks
3 remainder blocks
4 two-word Echo openings
11 middle-line Echo candidates
1 final return-word line
```

This is large enough to expose combinatorial and transition problems, but small enough for human review.

The goal is not maximum variety yet.

The goal is to discover whether a bounded library can produce variety without losing coherence or the selected Chamber world.

---

## 5. Resonance direction in the extracted blocks

The new gentle positive tendency is represented structurally rather than semantically.

Blocks may carry one or more gain tags such as:

```text
openness
orientation
answering
approach
visibility
held-open-relation
continuation
```

A future `CompositionPlan` must contain at least one gain allowed by `converging-return`.

Shadow material remains available:

```text
night
distance
dark
uncertain threshold
separate directions
```

But no selected route may terminate in:

```text
abandonment
entrapment
closed path
total extinguishing
erasure of one direction
return cancelling departure
```

The rule is therefore not:

```text
dark image forbidden
```

It is:

```text
shadow may shape the field
a generative direction must shape the movement
```

---

## 6. Word neutrality

Wish and return words appear only in reviewed grammatical positions.

Current wish-entry examples include:

```text
{wish_word} crosses the open frame

{wish_word} waits where distance begins to answer

Rain leaves room for {wish_word}

{wish_word} follows the light beyond the glass
```

Current remainder examples include:

```text
{return_word} remains where the passage stays open

{return_word} waits in the water between them

{return_word} keeps the shared direction visible
```

The blocks do not require a part of speech, article, inflection, plurality, or semantic category.

The visible display form will capitalise the submitted word once outside the library block selection.

---

## 7. Why some blocks carry more than one role

A few blocks carry both their primary role and `resonant_gain`.

For example:

```text
An open frame lets distance enter
```

establishes a field but also provides openness and approach.

Likewise:

```text
Two returns hold one interval open
```

creates the relation and supplies the required gain.

This is permitted only where the combined function has been reviewed explicitly.

The prototype should not infer roles from the visible sentence.

---

## 8. Echo extraction strategy

The Echo candidates are not a second independent poetry library.

They are constrained derivations that require visible material, block IDs, or operators from the completed long form.

For example:

```text
two returns hold one interval open
```

is available as a six-word Echo line only when `relation.interval.01` was selected in the long form.

Similarly:

```text
distance begins to answer
```

requires a long-form block that established that lexical and relational trace.

The future composer must still verify cumulatively:

- one sensory motif from the long form;
- one movement or relational operator;
- one concrete lexical trace;
- the wish word exactly once in line 2, 3, or 4;
- the return word exactly once as line 5;
- exact `2 / 4 / 6 / 4 / 1` word counts;
- no punctuation;
- no reversal of the verified resonance direction.

---

## 9. Deliberate repetition limits

Some phrases are memorable enough to become mannerisms if selected too often.

The first executable prototype should therefore record block IDs and later support at least simple limits such as:

```text
one block only once per composition
one conspicuous phrase family only once per composition
avoid the same relation block in immediate repeated test outputs when alternatives exist
```

No global user tracking or cross-device repetition memory is intended.

These limits concern only the construction of one local poem and controlled prototype sampling.

---

## 10. Questions for the first executable composer

The building-block library is ready to test the following questions:

1. Do the three structural routes produce genuinely different movement rather than shuffled paragraphs?
2. Are the profile requirements restrictive enough to preserve the selected world?
3. Are any relation blocks too explanatory when surrounded by simpler imagery?
4. Does every valid plan leave at least one resonant gain without sounding artificially optimistic?
5. Do word-neutral slots remain convincing with `Hope`, `Blue`, `Perhaps`, `Return`, and `Hush`?
6. Can the Echo inherit an actual lexical trace without becoming a miniature summary?
7. Which blocks become conspicuous after ten or twenty seeded outputs?
8. Which metadata fields are necessary for validation and which can be removed?

---

## 11. Next implementation boundary

The next step may now be a very small executable composer that supports only:

```text
mixed-threshold-return-world
```

It should:

- load the active profile world and this block library;
- accept injected randomness;
- select one valid route;
- filter before choosing each block;
- build an inspectable `CompositionPlan`;
- render one long form;
- derive one linked Echo;
- validate all structural invariants;
- print or return results without writing into the production Return flow.

It should not yet:

- modify the persistent opener;
- change the Return Artifact schema;
- claim support for the other two prototype worlds;
- claim coverage of all 125 combinations;
- move or archive V0.1 files.

The purpose of the first executable version is diagnostic:

```text
Can the reviewed library behave like a small poetry-building machine
without pretending that the machine understands poetry
```
