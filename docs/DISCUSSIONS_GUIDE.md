# Discussions Guide

GitHub Discussions are the public conversation space for glossai-nexus.

Discussions can be used for questions, feedback, resonance nodes, project orientation, and ideas that are not yet concrete code tasks.

Project discussions:

```text
https://github.com/NataliReise/glossai-nexus/discussions
```

## Preferred language

English is preferred so more people can participate.

German is also okay if English would make participation difficult. If useful, a short English summary can be added later.

## What belongs in Discussions?

Good topics for Discussions:

- questions about First Spark or the project idea,
- public-safe resonance nodes,
- feedback after trying a module,
- ideas for future modules,
- documentation suggestions,
- general project orientation,
- thoughts about public/private boundaries.

## What belongs in Issues?

Issues are better for concrete, actionable tasks.

Examples:

- a bug in First Spark,
- a broken link,
- a failing test,
- a concrete documentation fix,
- a small implementation task.

## Possible discussion categories

A small starting structure could be:

### Announcements

Project updates, stable milestones, and important notes.

### General

Open questions, impressions, and general orientation.

### Resonance Nodes

Optional public-safe resonance node posts.

### Ideas

Future module ideas, design thoughts, and possible extensions.

### Q&A

Practical questions about running First Spark, activation files, privacy boundaries, or contribution workflows.

The category list can stay small at first and grow only when needed.

## Resonance node posts

A resonance node is optional.

It should only show that a spark was seen, completed, or carried further.

It must not reveal private content.

A safe resonance node post may include:

```text
Resonance Node: N01-RN-draft
Module: Nexus 01 - First Spark
Status: completed
Trace visibility: public-safe summary only
Public alias:
Public note:

Consent:
I choose to share this public trace.
No private activation data, private gift text, or return artifact is included.
```

Only fill in `Public alias` and `Public note` if you want to.

## Never post private data

Do not post:

- `activation.local.json`,
- private gift messages,
- recipient-specific private data,
- return artifacts,
- private notes from a gift package,
- screenshots that reveal private activation content.

When unsure, leave it out.

## Contribution tone

This project is experimental and small.

Please keep discussions patient, careful, and public-safe.

Questions, corrections, and cautious experiments are welcome.
