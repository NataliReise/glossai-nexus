# Resonance Arc

The Resonance Arc describes how a private spark can travel without turning private meaning into public data.

It is a conceptual map, not a technical network yet.

## Core idea

A spark may travel privately.

A resonance becomes public only by consent.

A resonance node is not the gift itself. It is only a visible trace that a gift was seen, opened, or carried further.

## Layers

### Private layer

The private layer contains what belongs to a specific gift, moment, or person.

Examples:

- private activation files
- private gift messages
- private recipient data
- return artifacts
- private notes

This layer must not be posted publicly.

### Public-safe layer

The public-safe layer contains what can be shared without revealing private meaning.

Examples:

- public repository links
- neutral module descriptions
- public-safe resonance node drafts
- general feedback
- module ideas without private activation data

### Community layer

The community layer can grow through GitHub Discussions, issues, pull requests, and shared documentation.

Discussions are for questions, impressions, resonance nodes, ideas, and orientation.

Issues are better for concrete bugs or actionable tasks.

Pull requests are for proposed changes to public files.

### Future module layer

In the future, people may be able to create their own small Nexus modules.

Those modules should follow the same boundary:

- public module structure stays inspectable,
- private activation data stays private,
- public traces stay consent-based and public-safe.

## Resonance node

A resonance node is an optional public-safe trace.

It can show that a First Spark was seen or completed without revealing private content.

A resonance node may include:

- a public alias,
- a short public note,
- the module name,
- completion status,
- a public-safe trace statement.

A resonance node must not include:

- private activation messages,
- `activation.local.json`,
- private recipient data,
- return artifacts,
- private notes from a gift package.

Sharing is optional.

## Design principles

- The gift remains complete even if no public trace is shared.
- Passing on a spark is an invitation, not an obligation.
- Completion may create meaning, but it does not need to create files.
- Public visibility should be chosen, not automatic.
- The public project should make structure visible without exposing private content.
