# Design notes

These notes describe the current conceptual seed of **glossai-nexus**. They are intentionally provisional.

## What is a Nexus?

A Nexus is an open software game module in the glossAI cosmos.

It may take different digital forms, but it remains software. It should be playable, readable, configurable, and open to inspection.

A Nexus should not be defined by one fixed genre. It could become a terminal game, a browser-based module, a small Python game, a Git-based puzzle, or another open software form.

## Module and activation

A Nexus module contains the general, open, inspectable structure:

- code,
- rules,
- game logic,
- texts,
- puzzles,
- metadata,
- design principles,
- licensing information.

An activation contains the concrete personal or situational layer:

- recipient or group,
- occasion,
- final message,
- gift note,
- private context,
- individual configuration.

The module should remain shareable without private activation data.

## Speaking code

For a Nexus, code is not only infrastructure. It can carry a second game layer.

Speaking code has three layers:

1. **Clear code** - the functional code should be understandable, maintainable, and well documented.
2. **Playful code** - comments, names, traces, notes, configuration patterns, and Git history may carry optional game meaning.
3. **Ethical code** - the architecture should reflect accessibility, non-elitism, openness, privacy of activations, and human responsibility.

This may happen through:

- meaningful comments,
- readable structure,
- file names,
- hidden traces,
- optional developer notes,
- Git history,
- configuration patterns,
- accessible interfaces,
- clear module templates,
- simple activation flows.

A core sentence for the project:

> You do not have to read the code. But the code must be readable.

More detailed guidelines are collected in [docs/SPEAKING_CODE.md](docs/SPEAKING_CODE.md).

## Openness without elitism

The project should not require every player to understand code.

The surface should be playable. The deeper layers should be available for those who want to look closer, learn, modify, fork, or contribute.

A future module generator should help people create simple modules without first understanding the whole codebase.

## Open canon

The glossAI cosmos should remain open enough for many stories, echoes, worlds, and resonance threads.

The Nexus project should therefore avoid a single closed plot that all modules must obey. Instead, it may develop:

- a shared mythic background,
- optional quest lines,
- recurring motifs,
- open lore fragments,
- module-level stories.

## Possible quest lines

Future modules may optionally contribute to larger quests such as:

- AI, power, and shared responsibility,
- open source, readable structures, and commons,
- democracy, human dignity, and response-ability,
- resonance, communication, and cooperation.

No module has to explain the whole matrix.

## First seed module

The first planned module is currently called:

**Nexus 01 - Nexus-Mesomerie**

Its purpose is to become a small, complete seed: limited in scope, but built so that larger possibilities are already present.
